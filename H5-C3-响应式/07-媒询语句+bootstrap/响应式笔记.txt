1.1	t响应式基本介绍
响应式：所谓响应式页面追求的就是用户通过一个url 访问到的页面在不同的终端设备上都能看到一个很好的展示。

2 如何实现响应式
A 首先代码能做的事情就是100%的去还原设计稿，所以想要实现一个响应式网站就必须要有响应式的设计。
B 早期我们会利用 JS 来不停的监测当前屏幕尺寸的变化，然后根据尺寸来设置具做的CSS 样式
C 有了C3之后我们会采用它里面新增的媒体查询属性来快速简洁的实现上述的操作

1.2	C3媒询语句
在C3中定义了 @media 属性通过它我们可以对当前的媒体设备进行相应条件的查询。
@media 条件 {
	当上面的条件成立时会自动执行这里面的CSS样式
}
注意：
01 @meida 是CSS中的属性，虽然看起来用法和我们的 if 语句有点类似，但是二者有质的区别。
02 media 后面的大括号表示的是一个空间，在它里面需要书写具体的选择器和样式。
03 该语句定义的 CSS 样式仍然遵循之前的 CSS 语法规则，后写的一定会覆盖先写的，所以一般我们会将媒询样式写在默认样式的后面

1.3	媒询常用条件
01 max-width: 当宽度小于等于某一个值的时候会满足条件
02 min-width: 当宽度大于等于某一值的时候满足
03 orientation:Portrait|landScape 它是用来检测当前设备模竖屏的属性
landScape: 表示横屏
Portrait : 表示竖屏

1.4	媒询常用关键字
01 and : 可以用来连接多个条件，或者连接条件和设备类型。它左右二边的空格在使用时不能省略
02 only: 这个关键字的使用是从设备类型的角度上来考虑的，表示只选中某一种设备去声效
03 not : 和之前遇到的not 类似，表示排除的意思
04 all : 一般在工作中我们可以用这个关键字来表示选中所有设备类型

1.5	媒询样式的引入
01 可以直接将媒询样写在 style 标签中然后放置在 html文件里。
02 新建外部的 CSS 文件，如果需要适配多种设备，那么就定义多套样式，在这些CSS文件内部不需要书写 media 语句，
    最后可以直接通过 link 标签身上的 media 属性来实现同样的操作。

1.6	Bootstrap 基本介绍
Bottstrap 是由 twitter 公司推出一款基于 JQ 的前端响应式网站制作框架，在它的里面定义了 HTML CSS JS 的相应标准，
    我们在项目中可以直接引入相应的资源就可以快速的实现响应式页面开发。

1.7	Bootstrap使用步骤
01 将bootstrap中的资源引入到我们当前的项目中
02 在html页面中利用 link  script 引入我们需要的文件
03 在页面中书写我们需要的标签然后添加相应的类名就可以得到我们想要的结果

1.8	Bootstrap如何实现响应式
Bootstrap 的设计者根椐市面上主流的设备，经过分析然后将它们的尺寸分为了四个区间。分为用这四个区间来表示不同类的设备，
当屏幕尺寸大于等于1200px的时候定义版心为 1170,此类设备归为大屏桌面设备，尺寸大于996px 的时候定义版心为 970px 此类设备归不一般桌面设备，
尺寸大于768px的时候定义版心为750px 此类设备归不平板 pad ，小于768px 时都认为是手持设备，版心直接auto 布局。
在样式书写中通过 @media 语句来判断尺寸变化从而决定元素的显示样式。

1.9	Bootstrap中删格化布局
在bootstrap中有了上面的四个版心之后，为了方便的实现响应式布局，它又采用了一种删格化的思想，将当前设备屏幕宽度等分为12份，
这在给每一个元素写样式的时候就不去定义具体的 px 值，只会给它定义相应的列数就可以了。
比如说 col-lg-4 就表示当前元素在超大屏设备下占据4列，也就1/3布局，
与四个版心相对应的会有四个类前缀： col-lg- ? col-md-? Col-sm-? Col-xs-?

1.10	bootstrap常用的类名
1 container: 根据当前设备尺寸会自动定义宽度值的容器类
2 container-fuild:也是一个容器类，始终满屏显示
3 col-lg-pull-?: 表示在超大屏幕下让当前元素向左拉***列
4 col-lg-push-?：和上面一样表示向右推***列
5 btn | btn-lg | btn-primary : 分别表示按钮基础类，定义按钮大小类，定义按钮颜色类。
6 bg-danger: bg-就表示该类是定义背景色的类( background )
7 hidden-lg: 该类表示只在超大屏设备下才会隐藏
8 visible-lg: 该类表示只在超大屏幕下才会显示
注意：
只要上面出现了***-lg***-的地方都会对应着有四套类名规则。

1.11	云课常头问导航
01 navbar: 定义了一个通栏的导航条样式
02 navbar-inverse: 定义了一深色背景
03 navbar-fixed-top: 将当前元素固定定位在屏幕顶部
04 navbar-brand: 专门用存放logo图片和文字的盒子

06 navbar-toggle: 响应式导航条中最终的点击切换按钮
07 icon-bar    : 定义了一条白色的线条
08 nav : 它下面的 li 及 a 标签会有一定的样式
09 navbar-nav: 将它下面的 li a 做了一些适合在导航条中显示的样式
10 navbar-btn: 将它下面的btn 做了一些适合导航条显示的样式
11 pull-right: 定义了右浮样式
12 collapse : 将需要切换显示的内容放置在该容器中，默认是隐藏的
13 navbar-collapse: 定义了导航条中需要切换显示的内容默认是显示的
14 data-toggle=”collapse”: 该自定义属性表示需要开启切换功能
15 data-taraget=”选择器”： 指定了当前切换的目标元素

1.12	云课堂轮播图
01 carousel: 做为轮播图最大的容器内，里面管理三个模块。
02 slide: 定义了轮播图滑动动画的形式
03 data-ride=”carousel” : 表示开启轮播图的自动播放
04 data-interval=”” ：设置时间，单位是毫秒 表示每过多久切换一张图片
05 carousel-inner: 用来存放图片的大容器
06 item : 一个item 就可以存放一张图片，默认都是隐藏的。通过 active 进行显示
07 carousel-indicators: 圆点导航按钮的容器
08 data-slide-to: 设置一个编号，表示点击时跳转到第几张图片
